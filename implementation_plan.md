# Upgrading Petrol Radar to an Advanced Data Science & Analytics Platform

As the Principal Data Scientist for Premium Petrol Radar, I have outlined an advanced roadmap to transform this application from a real-time tracker into a highly intelligent, data-driven platform. We will implement predictive analytics, automated shortage detection workflows, and massive data aggregation for fuel usage and imports.

## Goal Description

Upgrade the project architecture to support heavy Data Science operations. This involves tracking historical usage, import volumes, daily sales, and running predictive edge-case workflows (e.g., forecasting shortages or demand spikes based on historical data).

## Proposed Architecture & Workflows

### 1. Data Aggregation Layer
We will expand our Supabase schema to ingest time-series data:
- **`fuel_imports_sales`**: Tracking national/regional imports of raw fuel (petrol/diesel) versus actual sales volumes.
- **`regional_fuel_usage`**: Tracking exact consumption metrics (litres sold) categorized by fuel grade (E20, XP100, Diesel, etc.) and location (City/State).
- **`fuel_pricing_history`**: Tracking historical price changes to build elasticity models.

### 2. Edge Case Prediction Engine (Automated Workflows)
We will build a predictive layer (e.g., via Supabase Edge Functions / Python backend) that automatically triggers alerts for specific edge cases:
- **Shortage Prediction**: If `sales_velocity` > `import_velocity` + `reserves`, flag regions for high probability of out-of-stock events (especially for rare fuels like Speed 97).
- **Price Hike Forecasting**: Analyze global crude import costs vs local retail pricing to predict incoming fuel price surges.
- **Holiday Demand Spikes**: Flag historical surge regions during long weekends and automatically notify the community.

## Proposed Schema Additions (`schema_v2_analytics.sql`)

This is the schema we will execute upon approval:

```sql
-- ── 1. Fuel Pricing History (Time-series) ─────────────────────
CREATE TABLE IF NOT EXISTS public.fuel_pricing_history (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  station_id BIGINT REFERENCES public.fuel_stations(id),
  grade_name TEXT NOT NULL,
  price_per_litre NUMERIC(8,2) NOT NULL,
  recorded_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_price_history_time ON public.fuel_pricing_history(recorded_at);

-- ── 2. Regional Imports & Sales (Macro Data) ──────────────────
CREATE TABLE IF NOT EXISTS public.fuel_imports_sales (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  region TEXT NOT NULL,
  fuel_type TEXT CHECK (fuel_type IN ('Petrol', 'Diesel', 'Aviation')),
  import_volume_kl NUMERIC(12,2) NOT NULL, -- in Kilolitres
  sales_volume_kl NUMERIC(12,2) NOT NULL,
  record_date DATE NOT NULL
);
CREATE INDEX idx_imports_sales_date ON public.fuel_imports_sales(record_date);

-- ── 3. Granular Fuel Usage (Micro Data) ───────────────────────
CREATE TABLE IF NOT EXISTS public.regional_fuel_usage (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  city TEXT NOT NULL,
  grade_name TEXT NOT NULL,
  daily_consumption_litres BIGINT NOT NULL,
  record_date DATE NOT NULL
);
CREATE INDEX idx_fuel_usage_city ON public.regional_fuel_usage(city);

-- ── 4. Edge Case Predictions (Model Outputs) ──────────────────
CREATE TABLE IF NOT EXISTS public.supply_predictions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  city TEXT NOT NULL,
  grade_name TEXT NOT NULL,
  shortage_probability NUMERIC(3,2) CHECK (shortage_probability BETWEEN 0 AND 1),
  predicted_exhaustion_date TIMESTAMPTZ,
  model_version TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
